# Note cleanup

Use this skill when the user wants an existing note formatted, corrected, clarified, or lightly cleaned up.

Purpose:
- Fix grammar and spelling lightly.
- Improve markdown structure and readability.
- Remove repetition, noise, and obvious filler.
- Add small missing headings or short clarifying additions only when they are directly supported by the note context.
- Preserve the user's meaning, personal context, examples, and useful nuance.

Do not:
- Add unsupported facts.
- Replace the user's intent with a generic summary.
- Delete useful context casually.
- Force a Zettelkasten, MOC, backlink, or PKM-heavy structure.

Required workflow:
1. Read the target note.
2. Identify the main issues briefly.
3. Propose a short cleanup plan.
4. Show the exact text, diff, or patch to add, change, or remove.
5. Ask for approval before editing.
6. Only edit after approval.

When unsure whether something is noise or useful context, keep it or ask before removing it.
